# Hypothesis Test Plan (План перевірки MVP)

## 1. Гіпотези

Гіпотеза 1 (зрозумілість керування):  
Якщо керування та стрільба інтуїтивні, то не менше 60% нових користувачів зможуть знищити ворога без пояснень протягом 2 хвилин.

Гіпотеза 2 (достатність базової механіки):  
Якщо базова механіка стрільби є цікавою сама по собі, то мінімум 3 з 5 користувачів оцінять досвід як «цікавий» або «зрозумілий» у короткому відгуку.

Гіпотеза 3 (технічна стабільність):  
Якщо реалізація MVP є технічно коректною, то під час 10 послідовних запусків не виникне жодного критичного збою або крашу.

## 2. План тестування

1. Користувачі: 5 тестових користувачів, новачків у грі.  
2. Процес тестування:  
   - Запустити гру.  
   - Пройти головний сценарій (знищення ворога).  
   - Спостерігати: час, запитання, кількість спроб, відчуття стрільби.  
   - Записати всі результати.  
3. Метрики для оцінки:  
   - % користувачів, що завершили сценарій без підказок  
   - Час проходження головного сценарію  
   - Кількість запитань  
   - Суб’єктивна оцінка користувачів  
   - Кількість критичних багів та крашів  
4. Інтерпретація:  
   - Якщо гіпотези підтверджені, MVP життєздатне.  
   - Якщо ні, визначити слабкі місця і скоригувати MVP.
